## 五、全栈联动：端到端推理性能分析

### 5.1 Decode 吞吐理论上限 (单卡, Batch=1)

|指标|公式|Llama-70B @H100 SXM (FP16)|
|---|---|---|
|每 token 需读取权重|`2 × params (BF16)`|~140 GB|
|HBM 带宽|—|3.35 TB/s|
|理论最大 Decode 速度|`带宽 / 权重大小`|3350 / 140 ≈ **~24 tokens/s**|
|实际 Decode 速度|(含 KV cache 读写等开销)|~18-20 tokens/s|

### 5.2 Prefill 吞吐理论上限 (计算密集)

|指标|公式|Llama-70B @H100 SXM (BF16)|
|---|---|---|
|每 token FLOPs|`2 × params`|~140 GFLOP|
|峰值 Tensor Core|—|989 TFLOPS|
|理论最大 Prefill 速度|`算力 / 每token_FLOPs`|989000 / 140 ≈ **~7,064 tokens/s**|
|实际 Prefill 速度|(利用率 ~50-60%)|~3,500-4,200 tokens/s|

### 5.3 推理关键性能指标 (Benchmark Metrics)

| 指标英文                             | 中文           | 说明                       |
| -------------------------------- | ------------ | ------------------------ |
| **TTFT** (Time to First Token)   | 首 token 延迟   | 受 Prefill 速度影响           |
| **TPOT** (Time per Output Token) | 每 token 生成延迟 | 受 Decode 速度影响            |
| **ITL** (Inter-Token Latency)    | Token 间延迟    | ≈ TPOT                   |
| **Throughput** (tokens/s)        | 吞吐量          | 全系统每秒输出 token 数          |
| **E2E Latency**                  | 端到端延迟        | TTFT + TPOT × output_len |
| **Goodput**                      | 有效吞吐         | 满足 SLA 的请求吞吐             |
