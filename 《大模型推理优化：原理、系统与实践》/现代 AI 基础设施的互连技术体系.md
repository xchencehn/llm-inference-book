

**从芯片内部到跨数据中心：现代 AI 基础设施的互连技术体系**

_2026 年 3 月_

---

## 引言：为什么互连技术决定了 AI 的上限

训练一个万亿参数的大语言模型，需要数万甚至数十万颗 GPU 协同工作数月之久。在这个过程中，每颗 GPU 不断产生梯度数据，并需要与其他 GPU 交换这些数据来完成参数同步。一个直觉性的事实是：如果 GPU 之间搬运数据的速度跟不上 GPU 计算的速度，那么更快的 GPU 并不会带来更快的训练——大量算力被浪费在"等待数据"上。

这意味着 AI 算力的真正上限，不是由单颗 GPU 的算力决定的，而是由 GPU 之间互连网络的带宽、延迟和效率共同决定的。互连技术是整个 AI 基础设施中最容易被忽视、却最可能成为瓶颈的环节。

本报告从底层物理原理出发，系统梳理现代 AI 基础设施中从芯片内部到跨数据中心的完整互连技术栈，剖析各层技术的设计逻辑、工程权衡、竞争格局与演进趋势。

## 第一章：RDMA——AI 互连的基础协议范式

### 1.1 传统网络通信的瓶颈

在 RDMA 出现之前，两台机器之间交换数据的标准路径是：应用程序发起系统调用，数据从用户空间拷贝到内核缓冲区，经过 TCP/IP 协议栈逐层封装，由网卡驱动交给网卡发出；接收端则反向逐层拆包，从内核缓冲区拷贝到用户空间。整个过程涉及多次内存拷贝、多次内核态与用户态的上下文切换、以及 CPU 全程参与的协议处理。每次数据搬运的延迟在数十微秒量级，CPU 开销显著。

对于 AI 训练中每秒数百万次的梯度同步操作，这种开销完全不可接受。

### 1.2 RDMA 的三大核心机制

RDMA（Remote Direct Memory Access，远程直接内存访问）通过三个机制颠覆了传统数据路径。

**零拷贝（Zero-Copy）**。应用程序注册的内存区域（Memory Region, MR）被锁定在物理内存中，网卡直接从该区域读写数据，不需要在用户空间和内核空间之间拷贝。这消除了传统路径中最昂贵的数据搬运操作。

**内核旁路（Kernel Bypass）**。应用程序通过用户态的 Verbs API 直接向网卡下发工作请求（Work Request），不需要经过操作系统内核。这消除了系统调用和上下文切换的开销。应用程序维护发送队列（SQ）、接收队列（RQ）和完成队列（CQ），直接与网卡硬件交互。

**协议卸载（Protocol Offload）**。可靠传输、确认应答、超时重传、排序、流控等协议逻辑全部在网卡硬件中执行，CPU 完全不参与数据面处理。执行这些功能的网卡被称为 RNIC（RDMA-capable Network Interface Card），它内部集成了完整的传输协议引擎、DMA 引擎和硬件队列管理逻辑。

启用 RDMA 后，数据路径缩短为：GPU 显存（通过 GPUDirect RDMA）→ RNIC → 网线 → RNIC → GPU 显存。端到端延迟降至 1-2 微秒，CPU 参与度接近为零。

### 1.3 RDMA 的历史演进：从 CPU 内存到 GPU 显存

RDMA 最初的设计目标非常明确——让一台机器的 CPU 直接读写另一台机器的主内存。最早的应用场景是高性能计算（HPC）和超级计算机集群。在技术路线上，它经历了三个阶段：InfiniBand（约 2000 年）作为最早也最成熟的承载网络，由 Mellanox 主导推动；iWARP（约 2007 年）尝试在 TCP/IP 上实现 RDMA 语义，但性能和复杂度不够理想；RoCE/RoCEv2（约 2010/2014 年）在以太网上直接运行 RDMA 协议（基于 UDP），兼顾性能与部署成本，成为数据中心主流方案。

转折点出现在深度学习和 GPU 计算的爆发期。GPU 显存中的数据要发到另一台机器的 GPU 上，传统路径需要先从 GPU 拷贝到 CPU 内存、再经 RNIC 发出，接收端再反向拷贝回 GPU，中间多了两次 GPU 与 CPU 内存之间的搬运。NVIDIA 通过 GPUDirect 系列技术逐步解决了这个问题：GPUDirect Peer-to-Peer（2011 年）实现同机 GPU 间 PCIe 直通；GPUDirect RDMA（2013 年，CUDA 5.0 + Kepler 架构）允许 RNIC 直接读写 GPU 显存；GPUDirect Storage（约 2020 年）进一步打通 NVMe SSD 与 GPU 显存的直连。

从架构视角看，GPUDirect RDMA 的本质是将 Memory Region 的注册范围从 CPU 主内存扩展到了 GPU 显存的物理地址空间。RNIC 并不关心内存在 DRAM 还是 HBM 上——它只需要一个有效的物理地址和 PCIe 通路即可完成 DMA。

## 第二章：以太网——互连世界的通用基座

要理解 AI 互连技术的竞争格局，必须先理解以太网——它是一切数据中心通信的基础底座，也是 InfiniBand 最大的竞争对手。

### 2.1 以太网的本质与定位

以太网（Ethernet）诞生于 1973 年，由 Bob Metcalfe 在 Xerox PARC 发明。它工作在 OSI 模型的最底两层——物理层（L1）和数据链路层（L2）。物理层负责比特信号在介质上的传输（电信号、光信号、编码方式、线缆规格）；数据链路层负责在直接相连的两个节点之间传输数据帧，定义了帧格式、MAC 地址寻址和介质访问控制。

以太网不关心 L3 及以上的事务——IP 路由、TCP 连接、HTTP 请求都是以太网之上的协议栈的事。以太网只负责把一个数据帧从一个 MAC 地址送到同一网络段内的另一个 MAC 地址。

### 2.2 帧结构

以太网帧是数据在线缆上的基本单元，以 IEEE 802.3 标准帧为例，其结构包括：7 字节前导码（Preamble，用于接收端时钟同步）、1 字节帧起始界定符（SFD）、6 字节目的 MAC 地址、6 字节源 MAC 地址、2 字节类型/长度字段、46-1500 字节有效载荷（Payload）、以及 4 字节帧校验序列（FCS，CRC-32 校验）。

帧的最小长度为 64 字节（不含前导码和 SFD），这与早期以太网的冲突检测机制直接相关——在 10Mbps 速率、2500 米最大网段下，信号最坏情况往返时间约 51.2 微秒，对应 512 比特 = 64 字节，帧必须足够长以确保发送方在冲突信号传回前仍在发送。标准 MTU（最大传输单元）为 1500 字节，但现代数据中心普遍启用 Jumbo Frame（MTU 放大到 9000 字节），以降低帧头开销占比、减少 CPU 处理帧的次数，对 AI 训练中大块梯度数据的传输尤为重要。

### 2.3 从共享总线到交换式全双工：以太网最重要的架构转变

早期以太网使用 CSMA/CD（载波侦听多路访问/冲突检测）协议：设备先"听"线路是否空闲，空闲则发送，若两台设备同时发送导致冲突，则各自等待随机时间后重试（二进制指数退避算法）。这是一个本质上"尽力而为"的竞争机制。

1990 年代中期的根本转变是交换机取代了集线器。集线器是物理层信号放大器，一个端口收到信号会无脑复制到所有端口。交换机工作在 L2，维护 MAC 地址表（CAM Table），只把帧转发到目标端口。当所有链路变为全双工的点对点连接后，冲突不再发生，CSMA/CD 在全双工模式下被禁用。这把以太网从"共享介质、尽力而为"的局域网技术，变成了可精确控制流量的交换网络平台。

### 2.4 物理层演进：从同轴电缆到 800Gbps 光纤

以太网物理层的演进是一部不断追求更高带宽和更远距离的工程史。

**同轴电缆时代**（1983-1990）：10BASE5 粗同轴电缆、10BASE2 细同轴电缆，总线拓扑，10Mbps。

**双绞线时代**（1990-2006）：10BASE-T（1990，Cat 3，10Mbps）→ 100BASE-TX（1995，Cat 5，100Mbps，4B/5B + MLT-3 编码）→ 1000BASE-T（1999，Cat 5e/6，1Gbps，PAM-5 编码配合回波消除和串扰消除，在四对双绞线上同时全双工传输）→ 10GBASE-T（2006，Cat 6a，10Gbps，PAM-16 + LDPC 纠错）。

**光纤时代**：10GbE（2002）→ 25GbE → 40GbE（4×10G）→ 100GbE（2010，4×25G）→ 200GbE（4×50G）→ 400GbE（2017，8×50G 或 4×100G）→ 800GbE（2024，8×100G）→ 路线图中的 1.6TbE。

推动高速化的关键技术包括：SerDes 单通道速率从 10G NRZ 演进到 100G PAM4（四电平编码在同波特率下翻倍比特率，代价是信噪比容限下降约 9.5dB）；FEC 前向纠错（如 RS(544,514)）成为高速链路的标配但引入额外延迟；光模块封装从 QSFP28（100G）演进到 QSFP-DD/OSFP（400G/800G），未来走向 CPO（共封装光学）。

### 2.5 L2 层关键机制

**MAC 地址学习与转发**：交换机通过自学习方式动态建立 MAC 地址表——从源 MAC 学习端口映射，查目的 MAC 决定转发或泛洪。这种即插即用的机制使以太网部署极为简便。

**VLAN（IEEE 802.1Q）**：在帧中插入 4 字节 VLAN Tag，包含 12 位 VLAN ID（1-4094），将交换机端口逻辑隔离为多个独立广播域。

**生成树协议（STP/RSTP）**：以太网帧没有 TTL 字段，环路中帧会无限循环引发广播风暴。STP 通过选举根桥并阻塞冗余链路来消除环路，但收敛慢且浪费冗余带宽。在现代数据中心，STP 已被 L3 到边缘 + ECMP 多路径路由或 VXLAN + EVPN 方案取代。

**流控机制**：802.3x PAUSE 帧是最早的全端口流控，但会导致队头阻塞。802.1Qbb PFC（Priority-based Flow Control）实现了按 8 个优先级队列分别暂停的能力，成为 RoCEv2 无损传输的基础。ECN（Explicit Congestion Notification）则在 IP 层配合拥塞控制算法（如 DCQCN）实现端到端的拥塞感知和速率调节。PFC + ECN 的组合是当前 RoCEv2 数据中心网络的标准方案，但 PFC 风暴和死锁问题使其在大规模部署中依然是最大的运维挑战。

### 2.6 以太网的根本特性

贯穿以太网 50 年历史的几个根本特性是：**"尽力而为"的传输模型**——不保证送达、不保证顺序、不保证不重复，只检测错误（CRC）然后丢弃，可靠性交给上层协议；**简洁的帧格式和无连接寻址**——48 位 MAC 地址、简单帧头、不需要建立连接，使硬件实现成本低且易于规模化；**持续演进的能力**——从 10Mbps 到 800Gbps 速率提升 10 万倍，但帧格式核心结构几乎不变；**开放生态**——IEEE 标准、多厂商竞争、开源软件栈，确保了供应链多元化和持续的成本下降。

正是这些特性，使得以太网即使在设计之初完全不是为高性能计算准备的，也能在 AI 时代通过持续的工程创新来挑战专门为 HPC 设计的 InfiniBand。

## 第三章：InfiniBand vs 以太网——AI 互连的两条路线

### 3.1 InfiniBand：为 RDMA 从零设计的专用网络

InfiniBand 不是在以太网上做改良，而是从物理层到传输层全部重新设计。它有自己专用的交换机、线缆、协议栈，与以太网是两个完全独立的世界。

InfiniBand 在链路层就内建了基于信用的流控（Credit-Based Flow Control）——发送方在没有接收方授予的信用时不会发送数据，因此理论上不会丢包。不丢包意味着不需要复杂的拥塞恢复机制，延迟非常稳定，尾延迟极低。其协议栈从 L1 到传输层浑然一体：链路层的信用流控保证无损，网络层的 LID 寻址精简高效，传输层的队列对机制直接映射到硬件。

代价是需要单独采购全套 InfiniBand 设备——专用交换机（NVIDIA Quantum 系列）、专用线缆、专用网卡（ConnectX 系列），且基本被 NVIDIA 一家垄断。

### 3.2 RoCE：在以太网上跑 RDMA

RoCE（RDMA over Converged Ethernet）的思路完全相反——不换网络底座，只换协议。RoCEv1 将 RDMA 帧直接封装在以太网帧中，受限于二层网络；RoCEv2 在 RDMA 帧外包一层 UDP/IP 头，可跨三层路由，成为当前主流。

RoCEv2 的工程本质是一个妥协：把 InfiniBand 的传输层协议（RDMA 动词、队列对、可靠传输机制）硬塞进以太网帧格式里。每个包额外携带 20 字节 IP 头 + 8 字节 UDP 头，这些对 RDMA 通信本身没有价值，纯粹是为了兼容以太网基础设施。在大包传输（4KB 以上）场景下，这 28 字节的开销占比不足 1%，影响较小。但在小消息场景（64 字节有效载荷），开销占比飙升到 17%，对消息速率和延迟产生显著影响。

更关键的问题不在帧头开销本身，而在以太网"有损"特性带来的连锁反应：RoCE 的可靠传输机制对丢包极其敏感，一旦丢包可能触发 Go-Back-N 重传（重传丢失点之后的所有包），性能断崖式下降。因此必须在以太网上叠加 PFC + ECN + DCQCN 来模拟 InfiniBand 链路层原生就有的无损保证，这套机制配置复杂且在大规模下容易引发 PFC 风暴和死锁。

### 3.3 核心差异对比

从可靠性看，InfiniBand 在链路层用信用机制天然保证无损，RoCE 依赖 PFC/ECN 等额外机制在以太网上"模拟"无损。从成本和生态看，RoCE 复用以太网基础设施，供应商多元，竞争压低价格；InfiniBand 是 NVIDIA 的封闭生态，成本显著更高（通常 2-3 倍）。从性能看，同等带宽下 InfiniBand 延迟更低更稳定，特别是 InfiniBand 独有的 SHARP 网内计算能力（将 AllReduce 卸载到交换机硬件执行）可带来 10-20% 的训练性能提升，这是以太网目前没有对等方案的能力。从规模看，InfiniBand 的子网管理器天然支持集中式路由管理，对大规模胖树拓扑友好；RoCE 依赖以太网分布式路由协议（BGP/ECMP），需要更精细的设计。

### 3.4 为什么不直接设计一个"开放的 InfiniBand"？

这个问题的答案不在技术层面，而在经济学和产业政治层面。

InfiniBand 本来就是一个开放标准。2000 年前后，Intel、IBM、Sun、Microsoft 等巨头联合成立了 IBTA，目标是用 InfiniBand 取代 PCI 总线成为通用互连。但 2002-2003 年，Intel 和 Microsoft 先后退出——Intel 转而推 PCIe 来保护 CPU I/O 互连的核心利益，Microsoft 决定押注以太网。大厂退出后 InfiniBand 萎缩到 HPC 小众市场，Mellanox 几乎成为唯一坚持的厂商。2020 年 NVIDIA 以 69 亿美元收购 Mellanox，InfiniBand 从"开放标准"事实上变成了 NVIDIA 的私有生态。

重新设计一个全新的 L2 协议在工程上也不现实：交换芯片需要全部重新流片（几十亿美元研发和 2-3 年时间）；所有网络操作系统、驱动、管理工具需要重写；全球数百万网络工程师的知识体系需要更新。重建一个完整 L2 生态的成本，远高于在现有以太网上打补丁的成本。

因此以太网阵营选择了渐进改良路线：保留以太网帧格式和生态外壳，逐步把 InfiniBand 的内核机制搬进来——先是 RoCEv2 + DCB（2010-2020），再是厂商私有优化如 Spectrum-X 自适应路由和 DSF 调度 Fabric（2020-2024），最终走向 UEC 1.0 标准化的信用流控、逐包喷洒和无连接传输（2025-）。

## 第四章：巅峰对决——两大阵营的前沿技术与真实部署

### 4.1 InfiniBand 阵营：NVIDIA 的全栈垂直整合

**Quantum-X800 + ConnectX-8（XDR 800Gbps）**是当前最新一代，提供 144 个 800Gbps 端口、端口到端口延迟低于 100 纳秒。路线图已明确下一代 GDR（1.6Tbps/端口）。

**SHARP v4（Scalable Hierarchical Aggregation and Reduction Protocol）**是 InfiniBand 的"杀手锏"。它把 AllReduce 等集合通信操作直接卸载到交换机硬件中执行——在传统 AllReduce 中每个 GPU 要把梯度发给所有其他 GPU，SHARP 让交换机在数据经过时直接做聚合计算，在树状层级中逐层聚合，大幅减少网络中的数据搬运量。SHARP v4 搭配 Quantum-X800 提供 14.4 TFLOPS 的网内 FP8 计算能力，报告的训练性能提升在 10-20%。

**自适应路由**在硬件层面实现逐包动态选路，实时感知拥塞并绕过热点，结合信用流控保持网络无损且负载均衡。**UFM 统一管理平台**提供全 fabric 实时遥测和 AI 驱动的预测性维护。

实际部署方面：Stargate（微软 + OpenAI）使用 XDR 800G InfiniBand 互联 64,000 台 GB200 系统（2025 年开始安装）；Oracle Zetta-scale 规划 131,000 颗 GB200 GPU 使用 Quantum InfiniBand；微软 Azure 的 ND 系列 HPC 实例长期使用 InfiniBand。

### 4.2 以太网阵营：百花齐放的多路线并进

以太网阵营不是铁板一块，而是多条技术路线并行推进。

**NVIDIA Spectrum-X**是一个有趣的存在——NVIDIA 自己做的以太网方案，本质上是把 InfiniBand 的优化技术移植到以太网上。Spectrum-4 SN5600 交换机提供 51.2Tbps 交换容量和 64 个 800Gbps 端口，配合 BlueField-3 SuperNIC 实现 RoCE 自适应路由和拥塞控制优化。xAI Colossus——100,000 颗 H100 GPU 组成的当时最大单一 AI 集群——用的就是 Spectrum-X 方案，网络吞吐效率达 95%。

**Meta 的 DSF（Disaggregated Scheduled Fabric）**走了完全不同的自研路线。核心创新是两域架构：网络分成以太网域和 fabric 域，在 fabric 域内数据包被拆成 cell，通过 VOQ + 信用流控 + 逐包喷洒在所有可用路径上均匀分布，出口处再重组为完整包。这本质上是把交换机内部的 crossbar 调度逻辑搬到了整个 fabric 层面，彻底绕过了 ECMP 哈希冲突的痛点。Meta 的集群规模已从 4K 扩展到 129K GPU。Meta 还开源了 CTran 和 NCCLX 传输层及集合通信库，直接挑战 NVIDIA NCCL 的主导地位。

**Google 的 Jupiter Fabric**是最极端的自研方案——连 GPU 都是自研的（TPU），网络使用 MEMS 光电路交换 + SDN 控制器，可根据流量模式动态重连物理拓扑，最新一代提供 13 Petabits/sec 双向带宽。

**字节跳动**自研了 veRoCE——一种 RoCEv2 兼容但能在有丢包和乱序的普通以太网上稳定运行的传输协议，直接消除了对 PFC 的依赖。

**Ultra Ethernet Consortium（UEC）1.0**于 2025 年 6 月发布正式规范（截至 2026 年 1 月已更新至 v1.0.2），代表了以太网阵营的统一标准。核心特性包括：无连接传输协议 PDC（解决 InfiniBand QP 可扩展性问题）、逐包喷洒（类似 InfiniBand 自适应路由）、信用流控 CBFC（从 InfiniBand 借鉴的无损机制）、包裁剪（拥塞时裁剪包头发送 NACK 触发重传，而非 PFC 暂停整个端口）。Broadcom 和 AMD 的首批 UE 硬件已开始出货，但大规模生产部署尚需时间。

### 4.3 真实部署全景

截至 2026 年初，大部分生产集群仍处于 400Gbps 这一代（InfiniBand NDR 400G 或 400GbE），800Gbps 正随 Blackwell 平台快速铺开。

关键部署包括：Stargate 选择 InfiniBand XDR 800G（64,000 GB200），Oracle Zetta-scale 选择 InfiniBand（规划 131,000 GB200），微软 Azure ND 系列使用 InfiniBand NDR 400G，xAI Colossus 使用 Spectrum-X 以太网 400G（100,000 H100），Meta 使用 RoCE + DSF（129,000 GPU），Google 使用自研 Jupiter + RoCE，字节跳动使用 RoCE + Scheduled Fabric + veRoCE，Amazon AWS 使用 EFA 自研以太网协议。

一个清晰的模式是：追求极致训练效率且不惜成本的部署选择了 InfiniBand（Stargate、Oracle），而追求规模、灵活性和供应商多元化的超大规模部署选择了以太网（Meta、xAI、Google、字节跳动）。

### 4.4 市场格局的转折

根据 Dell’Oro Group 2026 年 3 月的报告，以太网已在 AI 后端网络市场大幅超过 InfiniBand——2025 年 Q4 以太网在 AI 集群数据中心交换机销售中占比超过三分之二，体量超过 InfiniBand 的两倍。分析师明确指出：“不断增长的 AI 集群规模，加上持续的供应链约束，正在推动对供应商多样性的需求，因此推动了以太网的采用。”

趋势清晰：以太网在份额上已经反超，但 InfiniBand 凭借 SHARP 网内计算等独有能力在最高端的万亿参数模型训练中仍占据技术制高点。

## 第五章：NVLink——超越网络的内存互连

### 5.1 NVLink 的本质：不是网络协议，而是内存互连

NVLink 与以太网、InfiniBand 的根本区别在于抽象层次。后两者是"网络"——工作在"数据帧/包"粒度上，发送方封装、网络路由转发、接收方拆包。NVLink 是"内存互连"——GPU A 可以直接对 GPU B 的显存地址发出 load/store 指令，如同访问本地内存，没有帧头、没有路由、没有打包。NVLink 还参与缓存一致性——NVLink 连接的 GPU 之间，一个 GPU 修改了共享内存地址的数据，其他 GPU 的缓存会被自动失效或更新。这种语义是以太网和 InfiniBand 都不具备的。

NVLink 5（Blackwell 平台）的规格为：单链路 200GB/s 双向，每颗 GPU 18 条链路，合计 3.6TB/s 双向带宽——是 800Gbps InfiniBand（100GB/s）的 36 倍。

### 5.2 GB200 NVL72：72 GPU 全互连的工程实现

理解 NVL72 如何实现 72 GPU 的 all-to-all 互连，是理解 AI 集群架构的关键。

核心参数：每颗 B200 GPU 有 18 个 NVLink 5.0 端口；每颗第四代 NVSwitch 芯片有 72 个 NVLink 5.0 端口。NVL72 机柜包含 9 个 Switch Tray，每个 Tray 上 2 颗 NVSwitch，共 18 颗 NVSwitch 芯片。

连接拓扑的精妙之处在于：每颗 GPU 的 18 个端口恰好各连一颗 NVSwitch（每颗 NVSwitch 分配 1 个端口），反过来每颗 NVSwitch 的 72 个端口恰好各连一颗 GPU。这意味着每颗 NVSwitch 都"看见"了全部 72 颗 GPU。

当 GPU A 要与 GPU B 通信时，数据从 GPU A 发出到达某颗 NVSwitch，该 NVSwitch 在芯片内部做交叉交换，从连接 GPU B 的端口转发出去——只需 1 跳，不需要离开芯片。18 颗 NVSwitch 之间不需要互相通信，它们是 18 个完全独立的并行交换平面，作用纯粹是提供并行带宽。

数字验证：GPU 端口总数 = 72 × 18 = 1,296；NVSwitch 端口总供给 = 18 × 72 = 1,296。完美匹配，零超额订阅，完全无阻塞。

### 5.3 从 72 到 576：两层 NVLink 拓扑的扩展

72 颗 GPU 是当前 NVLink 架构的物理极限——因为一颗 NVSwitch 只有 72 个端口，必须全部直连所有 GPU 才能保持单跳全互连。

自然的扩展思路是加第二层 NVSwitch，形成两级 Clos 拓扑。NVIDIA 的路线图确认了这个方向：Vera Rubin Ultra NVL576（2027 年下半年）将 8 个 NVL72 机柜通过第二层 NVLink 互连形成 576 GPU 的统一域，再下一代 Kyber NVL1152 将达到 1152 GPU。

但扩展远非"立刻就能做到"。工程瓶颈包括：机柜间距离需要光互连（NVLink 信号速率极高，铜缆无法跨机柜），带宽在两层拓扑中无法保持完全无阻塞（跨机柜通信共享上联带宽，存在收敛比折中），NVSwitch 芯片需要升级以预留上联端口，576 GPU 统一域的功耗达 600kW 级别对供电和液冷提出极端要求，故障域扩大使运维复杂度显著增加。

### 5.4 为什么 NVLink 不能替代 InfiniBand/以太网

NVLink 的物理距离受限（铜缆 2 米以内，光互连仍在研发中），NVSwitch 端口数有限，且缺乏网络层的路由绕行和故障隔离能力。即使 NVLink 域扩展到 576 甚至 1152 GPU，10 万到 100 万 GPU 级别的集群仍需网络层互连。NVLink 和网络是互补关系——NVLink 提供域内超高带宽的内存语义互连，InfiniBand/以太网提供域间大规模扩展的网络语义互连。

## 第六章：分层互连架构——物理定律决定的最优解

### 6.1 为什么不存在统一的万能互连技术

不同通信场景对物理传输的要求差异极大：距离从毫米到千公里，带宽从 kbps 到 Tbps，延迟从纳秒到可容忍秒级，可靠性要求从容忍丢包到零比特错误。带宽和延迟与距离之间存在根本性的物理权衡，不存在一种技术能在所有尺度上同时做到最优。

### 6.2 四层互连体系

现代 AI 超级计算机的互连架构是一个明确的层级结构：

**第一层：GPU 内部**。技术为片上网络（NoC），带宽 TB/s 级，延迟纳秒以下，范围一颗芯片内。芯片内部模块间通信使用 mesh 或 ring 拓扑的硬件走线，没有任何协议开销。

**第二层：同节点内 GPU 到 GPU**。技术为 NVLink + NVSwitch，带宽 3.6TB/s per GPU（NVLink 5），延迟几十到一百多纳秒，范围 1 台服务器或 1 个 NVL72 机柜（72 GPU），未来可扩展到 NVL576（576 GPU）。

**第三层：节点到节点**。技术为 InfiniBand 或高性能以太网（RoCE/Spectrum-X/UEC），带宽 100-200GB/s per GPU（400-800Gbps 网络），延迟 1-5 微秒，范围数千到数万台服务器。

**第四层：集群到集群**。技术为以太网 + DWDM 光传输 + OTN，带宽 Tbps 级（聚合），延迟毫秒级（受光速限制），范围几十到几千公里。

每层带宽比上一层低约一个数量级，延迟高约一个数量级，但覆盖范围大数个数量级。AI 训练的软件栈（Megatron-LM、DeepSpeed 等）精确利用了这种层级结构：张量并行放在 NVLink 域内（最高带宽、最低延迟），流水线并行放在节点间（延迟敏感但带宽需求较低），数据并行放在最外层（带宽需求大但可与计算重叠）。

### 6.3 芯片间专用互连：以太网力所不及的领域

某些通信场景对带宽密度、延迟或语义的极端要求，远超以太网帧格式和 MAC 协议的设计能力：

**NVLink**提供 GPU 间内存语义的 load/store 操作，以太网只能做帧级数据传输。**Intel UPI / AMD Infinity Fabric**是 CPU 间维护缓存一致性的互连，要求深度集成 MESI/MOESI 协议，延迟几十纳秒。**CXL（Compute Express Link）**在 PCIe 物理层上提供缓存一致性和内存语义，让 CPU 与加速器共享内存池。**DDR/HBM**是最底层的内存接口。这些领域的共同特征是需要在"内存地址"甚至"缓存行"粒度上工作，以太网的帧级抽象在此完全不适用。

### 6.4 长距通信：以太网之外的世界

超过 10 公里的通信面临光信号衰减、色散、波长复用、电信级保护倒换等完全不同的物理挑战：

**SONET/SDH**是电信骨干的传统基石，基于同步时分复用，核心能力是 50ms 保护倒换。**DWDM**在单根光纤中同时传输几十上百个不同波长的光信号，每根光纤总容量可达几十 Tbps。**OTN**是取代 SONET/SDH 的新一代光传输容器，以太网帧作为其"客户信号"被装入容器在长距光纤上运输。**MPLS**在 L2.5 层为运营商骨干网提供标签转发和流量工程能力。

以太网在长距通信中的角色是被承载——OTN/DWDM 负责光传输，以太网帧是其中的货物。

### 6.5 以太网的"侵略"模式与边界

纵观整个 L1/L2 生态，以太网在持续扩张：Token Ring、FDDI 已退出历史舞台，SONET/SDH 在退缩，光纤通道在缩减，CAN 总线被车载以太网部分替代，CPRI 被 eCPRI 替代，InfiniBand 份额被蚕食。

以太网的扩张模式始终相同：先在核心领域积累巨大的规模优势和成本优势，然后通过"足够好 + 便宜得多 + 生态庞大"逐步取代相邻领域的专有技术。当专有技术的独有优势被以太网通过标准扩展（TSN 解决实时性、DCB 解决无损、UEC 解决 RDMA 级性能）逐步补齐后，成本和生态的碾压优势就让专有技术失去生存空间。

但以太网也有啃不下来的领域——芯片内部互连、CPU 缓存一致性、内存接口、无线空中接口。这些领域对带宽密度、延迟或物理特性的极端要求，远超以太网帧格式的设计初衷。

## 第七章：Clos 与 Fat-Tree——大规模互连的拓扑基础

### 7.1 核心问题：用小交换机拼大网络

现实中交换芯片的端口数有物理上限（当前主流 32-128 端口），无法直接造出一台连接数万设备的巨型交换机。1953 年贝尔实验室的 Charles Clos 解决了这个问题：用多台小交换机分级互连，通过多路径冗余，拼出大规模无阻塞交换网络。

### 7.2 三级 Clos 网络

标准 Clos 分三级：输入级、中间级、输出级。核心原则是输入级每台交换机都和中间级每台交换机有一条链路，中间级每台也都和输出级每台有一条链路。这样从任意输入端口到任意输出端口存在多条路径（经过不同的中间级交换机），中间级交换机数量足够多时可做到严格无阻塞。

### 7.3 Fat-Tree：带宽随层级增长的树

1985 年 MIT 的 Charles Leiserson 从树形网络角度提出了 Fat-Tree。普通树形网络越往上链路越少、带宽越窄，根节点成为瓶颈。Fat-Tree 的解决方案是"越靠近根节点的链路带宽越大"——通过在高层使用更多交换机和更多并行链路来"养胖"。

Fat-Tree 拍扁来看就是折叠的三级 Clos 网络，两个术语在工程实践中经常混用。学术上 Fat-Tree 强调"带宽随层级增长"的特性，Clos 强调"多级、多路径、无阻塞"的结构特性。

### 7.4 现代 AI 集群中的应用：Leaf-Spine

现代数据中心几乎都采用两级 Clos（Leaf-Spine）拓扑：Leaf 交换机（叶层）位于每个机架顶部，服务器直连其上；Spine 交换机（脊层）每台连接所有 Leaf，不直连服务器。任意两台服务器间的跳数一致（同 Leaf 下 1 跳，跨 Leaf 为 Leaf→Spine→Leaf 共 3 跳），通过 ECMP 多路径负载均衡。GPU 数量达到数万时加第三级 Core/Super-Spine。

Clos 拓扑的关键优势是非阻塞和均匀延迟。但 ECMP 基于五元组哈希选路径，对 AI 训练中的大象流效果差——这正是 DSF 逐包喷洒、Spectrum-X 自适应路由和 UEC 标准要解决的问题。

NVL72 和 NVL576 本质上也是 Clos 思想的应用：NVL72 的 18 颗 NVSwitch 是单级全交叉交换，NVL576 的两层 NVLink 拓扑是两级 Clos。同一个拓扑原则从 1953 年的电话网络贯穿到今天的 NVLink 交换架构——它优雅地解决了"用有限端口数的小交换机构建任意大规模无阻塞网络"这个根本问题。

## 第八章：光模块——AI 基础设施的新瓶颈与突破口

### 8.1 为什么光模块突然成为焦点

光模块（Optical Transceiver Module）做的事情很简单：电信号与光信号的双向转换。它并非新事物，但在 2024-2026 年成为资本市场和产业界的焦点，根本原因是两个趋势的交汇。

**需求爆炸**：10 万颗 GPU 的 AI 集群需要数十万个光模块，全球同时在建几十个这样的集群。仅 NVIDIA 的 GPU 订单就对应 700-800 万个 800G 光模块需求，整个市场在 2025 年超过 160 亿美元。

**铜缆撞墙**：这是更深层的物理原因。铜导线传输高速信号时，趋肤效应和介质损耗导致有效距离随速率急剧缩短——25Gbps/通道时铜缆可走 5 米，到 200Gbps/通道时不足 1 米，部分场景无法跨出单个机架。这就是行业所说的"铜墙"（Copper Wall）。AI 集群需要连接成千上万台服务器，距离从数米到数百米，铜缆已无法胜任，光纤从"可选的高端方案"变成了"没有它就建不了集群的刚需"。

### 8.2 三条激光器/光芯片技术路线

**EML（电吸收调制激光器）**是沿用二十多年的成熟路线。每个 EML 芯片在磷化铟（InP）衬底上集成激光器和调制器，800G 模块需要 8 个独立 EML 芯片。优点是技术成熟，问题是 InP 晶圆小（2-3 英寸）导致成本高、每个 EML 需要独立温控（TEC）导致功耗高、集成度低难以扩展。

**硅光子（Silicon Photonics）**是当前最大的技术变革。核心思想是在硅晶圆上用 CMOS 兼容工艺制造光学器件（调制器、光波导、光探测器）。硅的折射率（3.5）远高于二氧化硅（1.45），形成强光限制，波导截面可做到几百纳米。调制器靠载流子色散效应改变硅折射率，光探测器用异质外延锗层。硅光子芯片用 8-12 英寸晶圆线制造（面积是 2 英寸 InP 的 36 倍），波导位置由光刻定义（纳秒级精度，可用无源对准），多通道可共享激光源——综合带来集成度飞跃和成本结构变革。

硅光子的"阿喀琉斯之踵"是激光源：硅是间接带隙材料不能高效发光，必须依赖外部 InP 激光器芯片提供"光种子"。全球能做高性能 InP 激光器的厂商仅几家（Coherent、Lumentum、MACOM 等），产能在 AI 需求爆发前远未为此规模准备，成为当前供应链最紧缺环节。前沿解决方案包括异质集成（将 III-V 族薄膜键合到硅上）和微腔光频梳（一个泵浦激光器产生几十个均匀间隔波长）。

**VCSEL（垂直腔面发射激光器）**优势在于极低功耗和可晶圆级测试，适合短距（100-300 米）场景。Broadcom 正推 VCSEL-based NPO 方案用于机柜内短距互连。

### 8.3 三种架构形态的演进

**可插拔光模块**是当前主力。光模块做成标准 QSFP-DD/OSFP 封装插在交换机面板上，坏了直接拔换。问题在于交换芯片 SerDes 输出到模块笼子需要十几厘米 PCB 铜走线，在 200Gbps/通道下信号衰减严重，必须在模块内放 DSP 芯片做信号恢复，而 DSP 功耗占模块总功耗的 40-50%。一台 144 端口交换机仅光模块功耗即达 2000-3000W，超过交换芯片本身。

**LPO（线性可插拔光模块）**去掉模块内 DSP，将信号处理全部交给交换芯片 SerDes，每模块节省约 5-8W 功耗并降低延迟。2025-2026 年开始规模出货，是向 CPO 的过渡方案。

**CPO（共封装光学）**将光引擎直接集成到交换芯片同一封装基板上，铜走线从十几厘米缩短到几毫米。NVIDIA 声称 CPO 功效提升 5 倍、网络弹性提升 10 倍。NVIDIA 的 Quantum-X800 InfiniBand Photonics（CPO 版 InfiniBand 交换机）2025 下半年开始交付，Spectrum-X Ethernet Photonics 2026 下半年交付，Broadcom 的 Tomahawk 5-Bailly CPO 已量产。但大规模数据中心级部署预计 2028-2030 年才会真正铺开。CPO 的挑战在于光引擎与芯片共封装后故障维修复杂度增加，"可服务的 CPO"设计尚未完全成熟。

## 第九章：竞争格局与演进趋势

### 9.1 产业政治的核心张力

AI 互连技术的竞争本质上不是纯技术较量，而是产业控制权的博弈。选择 InfiniBand 意味着被 NVIDIA 全栈锁定——网卡、交换机、线缆、管理软件、乃至集合通信库（NCCL）全部来自一家供应商，没有议价权，没有供应链弹性。以太网阵营的核心驱动力正是摆脱这种锁定：Meta、Microsoft、AMD、Broadcom、Cisco、Intel 等巨头联合成立 UEC，本质上是一个反 NVIDIA 垄断联盟。

NVIDIA 自身也在两面下注——既推 InfiniBand 也推 Spectrum-X 以太网方案——这说明它清楚以太网的趋势不可逆，正在确保自己在以太网阵营中也占据重要位置。

### 9.2 技术差距正在缩小但尚未消失

InfiniBand 相对以太网的核心技术护城河在于两点：一是链路层原生无损带来的确定性低延迟，二是 SHARP 网内计算。前者正在被 UEC 1.0 的信用流控（CBFC）和 DSF 的 Scheduled Fabric 逐步补齐；后者目前仍是以太网阵营没有对等方案的独有能力——将 AllReduce 卸载到交换机中执行带来的 10-20% 性能提升，在万卡集群上意味着每月数百万美元的训练成本差异。

以太网阵营的优势在于：更低的 TCO（硬件成本约为 InfiniBand 的 1/2-1/3）、更灵活的供应链、更大的工程师人才池、以及更快速的创新迭代（多厂商竞争驱动）。对大多数工作负载而言，经过精心调优的以太网方案已能达到 InfiniBand 90-95% 的有效性能。

### 9.3 NVLink 域的持续扩大改变了竞争格局

一个值得深思的趋势是：NVLink 域从 8 GPU（DGX H100）扩展到 72 GPU（GB200 NVL72）再到 576 GPU（Vera Rubin Ultra NVL576），域内通信不再需要走网络。这意味着对"域间网络"的带宽和延迟要求实际上在下降——当 576 颗 GPU 组成的 NVLink 域可以独立完成大部分张量并行和流水线并行的通信需求时，域间只需要承载数据并行的梯度同步流量，而数据并行的通信模式更容易与计算重叠、对延迟不那么敏感。这客观上降低了以太网相对 InfiniBand 的性能劣势的实际影响。

### 9.4 光互连正在成为新的战略高地

从铜缆到光纤的转变不仅是物理层的技术换代，更是产业链的重构。光模块在 AI 基础设施中的成本占比正在快速上升——一个 10 万 GPU 集群的光模块采购成本可达数亿美元。硅光子和 CPO 技术的成熟度将直接决定下一代 AI 集群的规模上限和经济可行性。掌握光互连技术的供应商（Broadcom、Coherent、NVIDIA）在产业链中的话语权正在显著增强。

同时，CPO 技术的推进正在模糊"交换芯片"和"光模块"的边界——当光引擎与交换芯片共封装后，光互连不再是可替换的外围配件，而成为芯片不可分割的一部分。这对交换芯片厂商的能力版图提出了全新要求。

### 9.5 我的判断

综合技术演进趋势、市场格局和产业政治，我认为几个方向是比较确定的：

以太网在 AI 后端网络中的份额优势将继续扩大，但 InfiniBand 不会消失——它将退守到对性能和确定性有极端要求的顶级训练集群中，成为"高端小众"方案。这类似于光纤通道在存储领域的轨迹：份额持续萎缩，但在最关键的金融和医疗场景中长期存续。

UEC 标准的成熟是以太网阵营最重要的催化剂。当信用流控和逐包喷洒成为标准以太网交换机的原生能力后，以太网在大规模 RDMA 场景中的"先天不足"将从根本上得到弥合，RoCE 方案中最令人头疼的 PFC 配置和调优问题将被消除。

NVLink 域的扩大是一个渐进但深远的结构性变化。它不会替代网络互连，但会持续降低网络互连在整体训练性能中的权重——这进一步有利于以太网阵营，因为以太网的主要劣势恰恰在于网络层的性能差距。

光互连的"铜退光进"转变将在未来 3-5 年重塑数据中心的物理架构，CPO 从可插拔模块到共封装光学的演进将改变交换机的设计范式和产业链结构。硅光子的良率和成本曲线将是决定这一转变速度的关键变量。

## 附录：关键术语速查

**RDMA**（Remote Direct Memory Access）：远程直接内存访问，允许网卡直接读写远端主机内存，绕过内核协议栈。

**RNIC**（RDMA-capable Network Interface Card）：支持 RDMA 的网卡，内置传输协议引擎和 DMA 引擎。

**InfiniBand**：专为 RDMA 设计的全套网络协议和硬件生态，由 NVIDIA 主导。

**RoCE/RoCEv2**（RDMA over Converged Ethernet）：在标准以太网上运行 RDMA 协议的方案。

**NVLink**：NVIDIA 的 GPU 间高速内存互连技术，提供 load/store 内存语义。

**NVSwitch**：NVLink 的专用交换芯片，实现多 GPU 间的全交叉互连。

**SHARP**（Scalable Hierarchical Aggregation and Reduction Protocol）：InfiniBand 的网内计算技术，将 AllReduce 卸载到交换机。

**PFC**（Priority-based Flow Control）：以太网按优先级队列实现逐跳流控的机制。

**ECN/DCQCN**：以太网端到端拥塞控制机制。

**Clos 网络**：用多台小交换机分级互连构建大规模无阻塞交换的拓扑原则。

**Fat-Tree**：越靠近根节点带宽越大的树形拓扑，实际上是折叠的三级 Clos。

**UEC**（Ultra Ethernet Consortium）：推动以太网原生支持 AI/HPC 级互连能力的产业联盟。

**CPO**（Co-Packaged Optics）：将光引擎与交换芯片集成在同一封装内的光互连技术。

**硅光子**（Silicon Photonics）：在硅晶圆上用 CMOS 工艺制造光学器件的技术路线。
